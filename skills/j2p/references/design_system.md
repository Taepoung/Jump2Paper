## 컴포넌트 위치 (components.html)

### §1 — PAGE SHELL (9–757행)
전체 HTML 골격. 4단계에서 이 섹션 전체를 복사해서 새 파일의 시작점으로 사용.

---

### §2 — BASE COMPONENTS

| 섹션 | 행 | 목적 | 설명 |
|---|---|---|
| §2.1 히어로 | 764–775 | 논문 정체성 선언 | 제목, 저자, Venue, TLDR 등 핵심 메타데이터를 페이지 최상단에 배치할 때 (섹션 1 필수) |
| §2.2 수식 블록 | 777–805 | 수식 및 상세 해설 | 핵심 수식과 함께 변수 정의, 직관, 유도 과정, 선택 근거를 다층 패널로 설명해야 할 때 |
| §2.3 논문 인용 | 807–812 | 저자 원문 인용 | 아키텍처나 결과에 대한 저자의 핵심 주장이나 의도를 원문 그대로 강조하여 신뢰도를 높일 때 |
| §2.4 [보충]/[평가] | 814–817 | 배경 지식 및 주관적 인사이트 분리 | 논문 원문에 없는 기반 지식을 덧붙이거나(보충), AI가 비판적 관점으로 논문을 해석할 때(평가) 본문과 시각적으로 명확히 분리하고자 할 때 사용 |
| §2.5 callout | 819–826 | 강점 및 기여 강조 / 한계 및 전제 공유 | 논문의 독창적 기여점, SOTA 달성, 주요 장점 등 긍정적 측면을 부각할 때 / 논문의 가정(Assumptions), 작동하지 않는 케이스, 잔존하는 한계를 명시할 때 |
| §2.6 결과 테이블 | 828–850 | 정량 지표 비교 | 여러 모델이나 조건에 따른 수치적 성능 차이를 표 형식으로 대조할 때 (Ours/Baseline 하이라이트 활용) |
| §2.7 한계점 키워드 | 852–861 | 후속 연구 및 극복 방안 관련 탐색 유도 | 한계를 지적하는 것에 그치지 않고, 해당 한계를 극복하려는 최신 연구 동향(Google Scholar 키워드 등)으로 독자의 시선을 능동적으로 확장시킬 때 |
| §2.8 영향력 키워드 | 863–871 | 연구 기여도 및 연관 도메인 확장 | 논문이 끼친 학술적 파급력이나 관련 도메인의 주요 키워드를 제시하여 독자가 꼬리를 물고 추가 탐색을 이어가도록 연결 고리를 제공할 때 |
| §2.10 호버 툴팁 | 873–875 | 전문 용어 인라인(In-line) 정의 | 논문 내 낯선 고유 명사나 약어가 등장할 때, 독자의 읽는 흐름을 끊지 않고 마우스 호버만으로 즉각적인 의미를 파악하게 할 때 |

### §3 — VISUALIZATION & INTERACTIVE PATTERNS

각 섹션은 HTML + 인라인 `<style>` + `<script>` 로 자체 완결됩니다.
붙여넣은 후 논문에 맞게 내용(텍스트, 데이터, SVG 좌표)만 교체하세요.

| 섹션 | 행 | 목적 | 설명 |
|---|---|---|---|
| §3.1 블록 다이어그램 | 896–947 | 아키텍처 및 파이프라인 조감도 | 논문의 전체 구조, 데이터 입출력 포맷, 주요 모듈 간의 연결 흐름을 직관적인 블록과 화살표로 매핑하여 거시적 이해를 도울 때 |
| §3.2 계층형 드릴다운 | 949–996 | 복잡한 계층 모듈의 단계적 해부 | 단일 블록으로 표현이 불가능한 거대 아키텍처를 '전체 → 서브모듈 → 내부 연산' 단위로 숨김 없이 파고들며 메커니즘을 탐구할 때 |
| §3.3 데이터 플로우 | 998–1060 | 텐서 차원(Shape) 추적 | 데이터가 레이어를 통과하며 형태가 어떻게 변환되는지 추적하고, 차원 조작의 당위성과 트레이드오프를 한눈에 보여줄 때 |
| §3.4 스텝 플레이어 | 1062–1106 | 알고리즘의 동적 시퀀스 시연 | 복잡한 수학적 유도 논리나 의사코드(pseudo-code) 라인이 실제로 어떤 상태 변화를 만들어내는지 단계별로 쪼개어 재생할 때 |
| §3.5 수식 슬라이더 | 1108–1147 | 동작 원리 체감 (감도 분석) | 파라미터를 조작함에 따라 수식의 결괏값이나 형태가 실시간으로 변하는 모의 실험 환경을 제공하여 직관적인 깨달음을 유도할 때 |
| §3.6 히트맵 | 1149–1196 | 2차원 공간 상의 밀도 및 관계망 표출 | Attention 가중치분포, 상관계수, 단어 유사도 등 행렬 형태 데이터 속성 간 연관성 강도를 색상 농도로 뚜렷하게 대비시킬 때 |
| §3.7 비교 슬라이더 | 1198–1298 | Before/After 결과물의 직관적 대조 | 기존 베이스라인 모델과 제안 논문 모델의 결과물 이미지(생성 결과, 분할 마스크 등)를 슬라이더로 직접 문질러가며 비교 증명할 때 |
| §3.8 Chart.js 차트 | 1300–1333 | 다차원 성능 변화 양상 추적 | 에블레이션(Ablation) 기여도, 학습 에포크에 따른 곡선, 메트릭 분포 등을 선/막대 그래프로 그려 성능 개선 폭을 객관화할 때 |
| §3.9 임베딩 산점도 | 1335–1362 | 데이터 잠재 공간(Latent Space) 분석 | t-SNE나 PCA 등으로 차원 축소된 군집 데이터나 벡터 공간에서의 의미적 거리를 2D 평면 위에 가시적으로 흩뿌려 시각화할 때 |
| §3.10 노드-엣지 그래프 | 1364–1385 | 정보 전달망 구조 위상학적 시각화 | 그래프 신경망(GNN)의 구조, 지식 그래프 트리 검색(Tree Search) 등 요소(Node) 간 연관성에 묶인 복잡한 위상 구조를 표현할 때 |
| §3.11 행렬 시각화 | 1387–1426 | 연산 스텝 단위 병렬 행렬 변환 추적 | QKV 곱셈 단위 분할, 메모리 타일링(Tiling) 등 매우 구체적이고 세분화된 저수준 행렬 연산이 조립되는 물리적 계산 과정을 모식화할 때 |
| §3.12 분포 플롯 | 1428–1463 | 통계적 현상의 굴곡 및 윤곽 분석 | 확률 밀도 함수(PDF), 학습된 모델의 손실 함수(Loss Landscape) 표면, 노이즈 분포 등 확률/통계적 곡선 패턴 외형을 묘사할 때 |
| §3.13 메트릭 카드 | 1465–1481 | SOTA 달성 핵심 지표 대시보드 표출 | 정확성 향상, 추론 속도 단축 등 논문이 가장 자랑하는 결정적 수치 한두 가지를 별도 강조 카드로 뽑아내어 시선을 집중시킬 때 |

## 인터랙티브 요소 선택 가이드

**콘텐츠는 숨기지 않습니다.** 모든 내용을 노출하되, 인터랙션으로 탐색 깊이를 더하는 구조입니다.

| 논문 내용 | 사용 컴포넌트 | §위치 |
|---|---|---|
| 전체 모델 구조 | 블록 다이어그램 + 클릭 드릴다운 | §3.1 + §3.2 |
| 데이터 변환 과정 | 데이터 플로우 (사용자 제어 스텝) | §3.3 |
| 알고리즘 단계 / 수식 유도 | 스텝 플레이어 | §3.4 |
| 파라미터가 결과에 미치는 영향 | 수식 슬라이더 | §3.5 |
| Attention / 유사도 행렬 | 히트맵 | §3.6 |
| 기존 방법 vs 제안 방법 비교 | 비교 슬라이더 | §3.7 |
| 수치 실험 결과 / ablation | Chart.js 차트 | §3.8 |
| 벡터 공간 / t-SNE / PCA | 임베딩 산점도 | §3.9 |
| 계산 그래프 / GNN / 트리 | 노드-엣지 그래프 | §3.10 |
| 행렬 곱 / 어텐션 스코어 계산 | 행렬 시각화 | §3.11 |
| 확률분포 / 손실 분포 | 분포 플롯 | §3.12 |
| 핵심 수치 강조 | 메트릭 카드 | §3.13 |
| 기술 용어 정의 | 호버 툴팁 (`data-def`) | §2.10 |
